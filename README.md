政务信息采集平台（Scrapy 多站点）

技术栈：Python、Scrapy、Crawlab、XPath/Regex、Requests、python-docx、markdownify
项目描述：面向科技系统政府网站构建统一采集框架，覆盖新闻、通知公告等多栏目，抽取多类结构化字段，按标准目录归档上传。
核心职责与技术/业务落地：
•设计实现通用爬虫基类，统一列表页/详情页解析、日期提取、分页调度与异常兜底，提升代码复用性。
•基于配置化字段定义，实现多站点快速适配。
•实现增量抓取策略，按发布时间过滤+连续无新任务停止翻页机制，降低无效请求。
•开发1300+行自定义Pipeline，生成四类文件，支持附件下载、元数据组装及Crawlab数据上传。
•建立附件与反爬处理机制，识别异常场景并生成失败说明，保障结果可追溯与完整性。
•接入雪花ID方案，为各类文件分配唯一ID，保障多文件关联与后续入库一致性。
项目成果：
•形成“通用框架+站点配置”模式，新增站点仅需修改规则；兼容两种分页模式，输出结果兼顾结构化与原始留存，满足多场景需求。
